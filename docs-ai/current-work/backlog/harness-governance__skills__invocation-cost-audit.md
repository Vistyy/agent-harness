# Skill Invocation-Cost Audit

## Backlog Item

Audit high-frequency or broad entrypoint skills for unnecessary invocation cost
before splitting more skills.

## Current Bucket

Deferred Backlog.

## Scope

Evaluate:
- slimming `skills/verification-before-completion/SKILL.md` while preserving
  fresh-proof and quality-gate obligations
- compacting `skills/planning-intake/SKILL.md` without weakening decision
  closure, proof allocation, or deferral persistence
- whether `user-apps-design`, `mobile-design`, `webapp-testing`, and
  `mobileapp-testing` show repeated misroutes that justify a future `ui-routing`
  skill
- whether any completion-router split has evidence beyond possible prompt-size
  savings

## Required Evidence

Before proposing another split:
- name the flows that currently load too much context
- show the owner skill currently loaded and the smaller intended route
- identify invariants that must remain with the original owner
- reject splits that add another routing hop without observed friction
