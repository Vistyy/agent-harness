# uv

Use `uv` for Python work here.

## When

Use `uv` when project has:
- `pyproject.toml`
- `uv.lock`
- `requirements*` generated by `uv`

Do not switch tools inside Poetry or PDM projects.

## Rules

- Do not use raw `pip` in `uv` projects.
- Do not run plain `python`; use `uv run python`.
- Do not hand-roll virtualenv activation in `uv` projects.
- Do not introduce new `requirements.txt` unless legacy path clearly needs it.
- Use `uv tool install` only if user explicitly asks.

## Main Paths

### Project

Use when repo has `pyproject.toml` or `uv.lock`.

```bash
uv sync
uv add <pkg>
uv remove <pkg>
uv run <command>
uv run python -c "..."
```

### Script

Use for standalone Python files.

```bash
uv run script.py
uv run --with requests script.py
uv add --script script.py requests
```

### Tool

Use for one-off CLI tools.

```bash
uvx ruff
uvx pytest
uvx python@3.12 -c "..."
```

Only run trusted tool packages with `uvx`.

### Legacy pip Interface

Use only when project is clearly legacy and no `uv.lock` path exists.

```bash
uv venv
uv pip install -r requirements.txt
uv pip compile requirements.in -o requirements.txt
uv pip sync requirements.txt
```

## Quick Replacements

```bash
# Bad
pip install requests
python script.py
python -m venv .venv

# Good
uv add requests
uv run script.py
uv run <command>
```
