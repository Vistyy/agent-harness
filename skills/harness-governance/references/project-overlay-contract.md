# Project Overlay Contract

A harness-managed project keeps its local instructions as an overlay.

The overlay owns:

- product and domain facts
- roadmap and initiative queue
- architecture maps
- runtime topology and environment wiring
- project-local skills
- local command recipe bodies
- safety invariants that are specific to the project

The overlay must point to global harness owners instead of copying reusable
policy after the global owner is installed and proven.

Temporary duplicated project docs are allowed during an extraction wave only
when the active packet names their retirement or pointerization wave.
