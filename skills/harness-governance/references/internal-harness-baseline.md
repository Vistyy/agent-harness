# Internal Harness Baseline

Reusable baseline for harness-managed project enforcement.

Project checks should stay narrow, high-signal, and tied to owned policy
surfaces. Recurring reusable misses should move to global harness skills rather
than growing project overlays.

## Local Enforcement Shape

Project-local harness checks may enforce:

- wave brief and packet structure,
- delivery-map and backlog structure,
- durable cross-doc references,
- moved global surface resolution,
- test-layering and boundary-proof policy,
- project-specific runtime or command overlays.

Keep project-local checks as overlays when they depend on project paths,
project queues, or project runtime topology.

## Garbage Collection

When a local check proves reusable workflow, review, planning, delegation, or
proof doctrine, move the doctrine to the owning global skill and keep only the
project-specific check wiring locally.
